(1) What are the difference among map, filter, forEach and find in JavaScript?

Answer:
Map, filter, forEach, and find are four different array methods in JavaScript, each with their own unique use cases:

Map: Allows transforming each element of an array into a new array of the same length. Map returns a new array with the same number of elements as the original array, but with each element transformed according to the function provided.
Filter: Allows selecting a subset of elements from an array based on a provided condition. Filter returns a new array with only the elements that satisfy the condition.
forEach: Allows iterating over each element in an array and performing a specified action for each element. forEach does not return a new array and simply executes a provided function for each element.
Find: Allows finding the first element in an array that satisfies a provided condition. Find returns the first element in the array that meets the condition, or undefined if no element is found.

In summary, map is used to transform each element of an array into a new array, filter is used to select a subset of elements based on a condition, forEach is used to execute a function for each element in an array, and find is used to find the first element in an array that meets a condition.

(2) What are the difference among localStorage, sessionStorage and cookies in JavaScript?

Answer:
localStorage, sessionStorage, and cookies are three different ways to store data in JavaScript, but they have some key differences:

localStorage: Allows storing larger amounts of data (up to 5-10 MB) that will remain even after the browser is closed or the computer is turned off. The data is stored indefinitely or until it is manually cleared by the user.
sessionStorage: Similar to localStorage, but the data is only available for the duration of the session. This means that the data is lost when the browser tab or window is closed.
Cookies: Allows storing small amounts of data (up to 4 KB) that can be accessed by both the server and client-side code. Cookies can be set with an expiration time, after which they will be deleted. They can also be restricted to a specific domain or path.

In summary, localStorage is ideal for long-term storage of larger amounts of data, sessionStorage is best for temporary storage, and cookies are useful for storing small pieces of information that need to be shared between the server and client-side code.

(3) What is event loop in JavaScript and how does it work?

Answer:
JavaScript is a single-threaded language but it can also handle asynchronous operations using the event loop. The event loop is a mechanism that allows JavaScript to run code, process events, and execute queued tasks in an orderly fashion. The call stack is where JavaScript executes its code, while the callback queue is where it stores functions that are waiting for execution.

The event loop constantly checks for code on the call stack and moves any function that's ready to be executed from the callback queue to the call stack.
There are two types of tasks: microtasks and macrotasks, with the event loop prioritizing microtasks over macrotasks. This means that after every macrotask execution, it will run all microtasks in the queue before moving on to another macrotask. This ensures that microtasks don't get delayed by long-running macrotasks and can update the state of the application quickly.
The event loop is what makes JavaScript powerful and versatile. By leveraging its features, we can create responsive web applications that can handle complex logic without compromising user experience.

In summary, JavaScript can handle asynchronous operations using the event loop, despite being a single-threaded language. The event loop allows JavaScript to process events and execute queued tasks in an orderly fashion, with microtasks being prioritized over macrotasks. By leveraging the event loop's features, developers can create responsive web applications without compromising user experience.

(4) What are the functionalities of quizTimer function in noChange.js file?

Answer:
quizTimer function is a timer function that runs a quiz.

The function takes one parameter, which is called "dismiss". If "dismiss" is true, it clears the timer and stops the quiz.
When Submit Quiz button is pressed, at first it checks if all the answers are selected in the quiz. If (answers.length == 6) which means all answers are selected, then quizTimer(true) is set to true on click event listener to stop the timer. This is how "dismiss" is set to true to stop the timer.

Until "dismiss" is set to true, this function continues the timer and updates the quiz timer display every second. The timer function increments a count variable every second, converts the count variable into minutes and seconds, and updates the HTML on the page to display the time consumed for the quiz.

Additionally, if the count is greater than 60 seconds, the function alternates the text color of the timer display every second to indicate that the consumed time in quiz is more than 1 minute.

In summary, quizTimer function is a timer function that runs until the Sumbit Quiz button is pressed after all the quiz answers are selected (if all answers are not selected the timer will continue). It also updates the HTML by showing the consumed time for the quiz and if the time is more than 60 seconds, it alternates the text color to indicate that the 1 minute limit has passed.

(5) What are the functionalities of chooseQuiz function in noChange.js file?

Answer:
This is a function named chooseQuiz that takes two arguments: index and givenAns.
Here, index indicates the serial number of the selected question and givenAns is the option selected by the user.

The purpose of this function is to update an array of quiz answers called answers based on the givenAns provided by the user.
This function is triggered when clicking on any of the generated options during the quiz.

In this function, in the first line isExist variable uses the find method to search the answers array for an object that has the same id value as the question being answered (quizData[index].id).

If the answer of a question is already given by the user (isExist is true in that case), the function uses the splice method to replace the existing quiz object with a new object that combines the quizData[index] object with the givenAns value provided by the user.

If the answer of a question is not given yet, the loop continues until it reaches the end of the answers array.

If isExist is false (that means user hasn't given the answer of that question yet), the function enters an else statement and adds a new object to the answers array that combines the quizData[index] object with the givenAns value provided by the user.

In summary, this function checks if a quiz answer already exists in the answers array and updates it if it does, or adds a new answer to the array if it doesn't. It then calls a separate function named displayAnswers() by passing the answers array as parameter to display the updated answers to the user.
